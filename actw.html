The Box Model and Display Properties

What is the Box Model?

Just a way of saying: "Every element in web design is a rectangular box." The box has certain properties that we can set, which we will go over shortly.

Why does this Matter?

It may seem obvious, but boxes are useful shapes, they fit together nicely next to and inside of each other and have a width and a height. Using HTML and CSS we can define those widths and heights and piece together these boxes of content to build a webpage. Before we talk about the details of the box model, it's important to understand the different ways HTML elements can be displayed, however.

Understanding the Display Property

The three most important display properties in CSS: block, inline, and inline-block. Let's go over them!

Display: Block

An element with display: block will occupy all available width, regardless of content, and begin on a new line. We call these block-level elements.

Tags that default to being block-level include p, div, section, header, footer, and all heading tags (like h1, h2, etc.)

Display: Inline

An element with display: inline will occupy only the width of its content and does not begin on a new line. It will not respect any custom widths and heights that you set, though it does have an inherent width and height based on its own content. It will respect padding declarations but not top and right margins (more on that in a bit).

Defining Margin, Padding, and Borders

There are a couple ways to define margins, borders, and padding. The first is to explicitly set them: 


.mystyle {
  margin-left: 10px;
  margin-right: 5px;
  margin-top: 30px;
  margin-bottom: 20px;
}

Then there is a shorthand way to define them all in one property. The four spots in this single-line declaration always correspond to TOP RIGHT BOTTOM LEFT: 


.mystyle {
  margin: 30px 5px 20px 10px;
}

Defining Margin, Padding, and Borders

Often you will find yourself defining a property where the top and bottom margins/padding/borders will be the same and then the left and right ones will be the same as each other but different from the bottom and top. There's a shorthand way to do this: 


.mystyle {
  margin: 10px 20px
}

So in the above example margin-top and margin-bottom are 10px and the margin-left and margin-right are 20px.

Putting it Together: Display Inline-Block

An element with display: inline-block is (as you can imagine) a hybrid between display block and inline. They do not sit on their own line, but they will respect top and bottom margins declarations and width and height declarations, and like display: inline will also respect padding and margin left and right declarations.
